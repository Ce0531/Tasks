# 1️⃣阶段一
## 1.GitHub上不去问题  
1. 原因  
- 校园网通常对境外网站带宽限制更严格，路由绕路多，延迟高、丢包率大  
- 国内普遍存在的 DNS 污染、GFW 干扰，也会导致 GitHub 连接不稳定或无法访问  
- 本地网络环境（如家庭宽带、移动网络）也可能因运营商出口拥堵出现类似问题
2. Watt Toolkit作用
- 本质：它是网络加速工具，并非单纯“提升网速”，而是优化访问境外服务的连接路径与稳定性  
3. 思考  
   - DNS污染是篡改解析结果，让你找不到正确服务器；
   - GFW是国家级流量拦截系统，从IP、端口、内容多维度阻断跨境访问；
   - Watt Toolkit本质是通过绕开污染DNS、选择优质路由的方式，在现有网络规则下提升连接稳定性。
## 2.链接问题  
1. 问题：原本Task的内容没有分层，也无链接可点，一点都不清晰，网络上找教程也没找到适合的
2. 解决：多次点开mentor的主页，学习模仿他的界面，并向AI提问，最终做出分层链接   
## 3.DNS污染 💻
1. 发生机制：DNS污染是一种中间人攻击，攻击者在DNS查询响应返回前，抢先发送伪造的DNS解析结果。当客户端接收后，会将错误的IP地址缓存，后续访问该域名时都会指向错误服务器。
2. 污染对象：主要污染**DNS缓存**（本地缓存、递归服务器缓存）和**DNS解析响应包**，让域名解析到恶意或不可用的IP地址，从而阻断或劫持访问。   
## 4.GFW 🌐
1. 定义：GFW（Great Firewall，中国国家防火墙）是一套部署在国家级网络出口的流量监控与过滤系统，用于管控跨境网络访问。
2. 干扰方式：
   - IP封锁：直接封禁特定境外服务器IP。
   - DNS污染/劫持：篡改DNS解析结果，使域名无法解析到正确IP。
   - 关键字过滤：检测数据包中的敏感内容，主动发送RST包中断TCP连接。
   - 端口封锁：限制特定协议端口（如部分VPN端口）的跨境流量。  
## 4.Watt Toolkit 优化原理 🛠️
 Watt Toolkit（原Steam++）的“优化连接路径”主要通过以下操作实现稳定访问：
 - 修改Hosts文件：手动绑定域名与优质IP地址，绕过污染的DNS解析，直接连接到可用服务器。
 - 智能选路：测试并选择延迟更低、丢包率更小的网络路由节点，避开拥堵或被干扰的链路。
 - 本地代理/端口转发：通过轻量代理工具，将应用流量转发到更稳定的出口节点，减少被GFW干扰的概率。
 - 缓存优化：缓存解析结果与连接状态，避免重复建立连接，提升访问稳定性。

# 3️⃣阶段三    
## 1.网页 Favicon 图标设置后不显示
 - 原因：浏览器缓存强制加载旧图标；图标格式不规范
 - 解决：改为相对路径 href="favicon.png"；使用 Ctrl+F5 强制刷新；使用标准 ico/png 格式   
## 2.视频默认显示在页面中间，无法置顶
 - 原因：默认 inline布局有间距，未清除默认边距
 - 解决方案：设置 `video { display: block; }`，配合 `position: absolute; top: 0; left: 0; width: 100%;`实现置顶  
## 3.角色卡片用 float 布局，出现错位、重叠  
 - 给`.char-box`设置 `width: 48%`(自己试)，预留间距；添加`.clear `类 `div`，设置 `clear: both` 清除浮动；固定角色图片尺寸，统一卡片高度  
## 4.固定导航栏（`position: fixed`）遮挡下方板块内容  
 - 给所有`.section `类添加` padding: 80px 20px`，顶部预留导航栏位置(以后要根据情况改)  
## 5.视频播放一遍消失，标题显现  
 - 用JS监听播放结束 `ended`事件，通过 CSS `opacity`实现视频淡出、标题淡入  
 - JS监听效果还没学到，`opacity`我也没学过，故没有用到网页上，后续可学习相关内容实现  
## 6.在VSCode新建了文件，只能提交到本地仓库，不能同步远程GitHub  
 - 关联远程仓库：`git remote add origin 仓库地址`  
 - 第一次推送并绑定分支：`git push -u origin main`
 - 后续直接在VS Code点同步或推送

# 4️⃣阶段四  
## 1.Q&A 折叠面板点击展开异常  
 - 问题：点击问题标题后，回答区域仅轻微下移，无法完整显示内容；多次修改样式后仍出现展开不完整、动画失效、无响应等情况  
 - 原因：CSS过渡属性与实际变化属性不匹配使用`transition: transform`却修改`height`，导致浏览器无法识别动画。`height: auto`不支持CSS过渡,CSS 无法对`0 → auto`进行插值计算，导致高度无法正确撑开。JS与CSS控制逻辑冲突,JS直接控制`display`，CSS控制`height/max-height`，两套规则互相覆盖。不合理内边距导致内容挤压,展开状态设置过大`padding`，使内容区域被压缩或超出可视范围    
 - 解决：JS 只负责切换类名，CSS 负责所有样式与动画
      
 - css    
  ```css
  .qa-a {
  max-height: 0;
  overflow: hidden;
  transition: max-height 0.4s ease, padding 0.4s ease;
  padding: 0 20px;
  }
  .qa-item.active .qa-a {
  max-height: 800px;
  padding: 20px;
  }  
  ``` 

 - js  
  ```javascript  
  const items = document.querySelectorAll('.qa-item');
  items.forEach(item => {
    item.querySelector('.qa-q').addEventListener('click',    () => {
      item.classList.toggle('active');
     });
   });   
   ```
## 2.CSS 定位与悬浮按钮布局  
用CSS做返回顶部按钮  
 - HTML  
  ```html  
  <div id="top"></div>
  <a href="#top" class="go-top">↑</a>   
  ```  
 - CSS  
  ```css  
  .go-top {
  position: fixed;
  bottom: 30px;
  right: 30px;
  width: 50px;
  height: 50px;
  border-radius: 50%;
  background: #3b82f6;
  color: #fff;
  display: grid;
  place-items: center;
  text-decoration: none;
  font-size: 22px;
  z-index: 999;
  transition: 0.3s;
  }
  .go-top:hover {
  transform: scale(1.1);
  background: #1d4ed8;
  }  
  ```  
## 3.class 与 id 选择器优先级与样式覆盖  
 - 问题：部分样式不生效，控制台查看样式被划掉，出现样式覆盖  
 - `!important` > 行内样式 > ID 选择器 > 类 / 属性 / 伪类 >  标签选择器  
## 4.DOM加载顺序导致JS获取不到元素  
 - 问题：控制台报错：无法读取 null 的属性，事件绑定无效   
 - 解决：将`<script>`放在`</body>`结束标签前  
 - 操作DOM必须保证节点已生成    
## 5.盒模型导致布局宽度计算错误  
 - 原因：默认盒模型`box-sizing: content-box：`  
 - 实际宽度 = width + padding + border  
 - 解决：  
   ```css  
   * {
     box-sizing: border-box;
   }  
   ```  
## 6.页面平滑滚动  
 - 
   ```css  
   html {
     scroll-behavior: smooth;
   }  
   ```  
 - 无需JS，全站锚点跳转自动平滑  
 - CSS平滑滚动优先级低于`JS scrollTo`
