# 1️⃣简述提交流程
## 1. 关于Git提交练习文档（通过Git Bash上传）  
   - **准备阶段**   
     进入项目目录：cd/文件地址
     初始化：git init  
   - **创建文件**  
     生成文件：echo "# Hello,曾新颜" > 文件名  
     验证：cat 文件名  
   - **暂存提交**  
     暂存：git add 文件名  
     提交到本地仓库：git commit -m "说明" 
   - **推送至远端**  
     关联远程仓库：git remote add origin 地址  
     推送到main分支：git push origin main  
## 2. 关于其他提交方法  
   - **前情提要**：开始打算用VSCode，教程都看完了，结果发现要求是用Bash提交。用Bash提交时老是打错，加上刚开始用没上手,导致提交一个简单的Hello都花了很长时间,还发生了远程冲突。所以上传完立马抛弃Bash，转战VSCode.
   - **VSCode使用感受**：可以下载插件，功能也很多，容易上手，所见即所得，除了Markdown还能写代码，很方便。
   - **GitBash是否麻烦**：GitBash命令行需要记忆指令、手动输入路径，对新手有一定学习成本，且纯文本操作不够直观，容易因拼写错误导致失败。但它稳定性强、可自动化程度高，适合熟悉后高效批量操作。
   - **其他方法**  
    1. IDE 集成工具：VS Code、IntelliJ 等编辑器内置 Git 可视化面板，可直接点击「暂存」「提交」「推送」，无需记忆命令，所见即所得。  
    2. GUI 客户端：GitHub、 Desktop等，用图形界面展示分支、提交历史，拖拽即可完成操作，适合新手快速上手。     
    ​3. 脚本/别名：将高频操作封装为 Shell 脚本或 Git 别名，简化命令输入。
# 2️⃣学习笔记  
1. 关于Git  
   Git 是一个分布式版本控制系统，核心作用是：
   - 追踪代码/文件的每一次修改，记录谁、在何时、做了什么改动   
   - 支持多人协作开发，避免代码冲突   
   - 管理项目的版本演进，可随时回滚到历史版本  
   - 本质上管理的是文件的变化快照，而非完整文件副本，因此高效且轻量  
2. 四大核心区  
   |区域|定义|  
   |---|---|  
   |工作区|本地电脑上直接编辑文件的目录，是可见的文件状态|  
   |暂存区|临时存放待提交的修改，是  git add  后文件的过渡区域|  
   |本地仓库| .git  目录下的版本库，存储所有提交的历史和元数据|  
   |远程仓库|托管在服务器（如 GitHub/GitLab）的仓库，用于多人协作和备份|  
3. 完整流转过程  
   - **git add <文件名>**  
​   a. 作用：将**工作区**的修改添加到**暂存区**  
​   b. 变化：文件从「未跟踪/已修改」状态，变为「已暂存」，准备进入版本记录  
​   c. 场景：可以多次  add ，逐步把要提交的修改都放进暂存区      
   - **git commit -m "提交说明"**   
​   a. 作用：将**暂存区**的所有修改永久保存到**本地仓库**，生成一个新的版本快照  
​   b. 变化：暂存区内容被清空，本地仓库新增一条提交记录（包含作者、时间、修改内容）  
​   c. 场景：完成一个功能/修复一个 bug 后执行，提交说明要清晰描述本次改动    
   - **git push <远程名> <分支名>**    
​   a. 作用：将**本地仓库**的提交同步到**远程仓库**  
​   b. 变化：远程仓库更新为本地最新版本，其他协作者可以拉取到你的代码  
​   c. 场景：完成阶段性开发后推送到远程，方便团队共享和备份    
4. 操作笔记
   - 克隆远程仓库到本地：git clone <地址>   
   - 初始化（从0开始创建时）：git init    
   - 将文件添加到暂存区：git add <文件>   
   - 生成本地版本记录：git commit -m "说明"   
   - 绑定远程仓库（clone后自动完成）：git remote add origin <地址>  
   - 推送代码至远端：git push origin main   
   - 拉取远程最新代码：git pull origin main  
   - 查看当前仓库状态：git status  
   -  查看提交历史：git log  
# 3️⃣【进阶】版本回滚和冲突解决  
## 1.版本回滚  
   - **未推送到远端**  
     保留代码，撤销提交：git reset --soft HEAD^   
     丢弃代码，彻底撤销：git reset --hard HEAD^  
   - **已经推送**  
     git revert  
     git push origin main
