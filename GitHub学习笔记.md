# 1.概念  
|概念|英文|含义|  
|---|---|---|  
|仓库|Repository|存放项目代码、文档的独立空间，是 GitHub 最基础单元|  
|星标|Star|收藏感兴趣的项目，方便后续查看，也代表项目受欢迎程度|  
|复刻|Fork|复制他人仓库到自己账号下，可独立修改，不影响原项目|  
|议题|Issue|用于提交 bug、功能建议或讨论，是项目协作的沟通渠道|  
|拉取请求|Pull Request |向原仓库提交代码修改，请求合并到主分支，是开源协作核心流程|  
|分支|Branch|独立于主代码的开发线，用于功能开发、bug 修复，避免影响稳定版|  
|合并|Merge|将一个分支的代码变更整合到另一个分支（如开发分支 → 主分支）|  
# 2.Fork与协作流程  
1. 在目标仓库页面点击Fork，复制到自己账号  
2. 克隆自己的Fork仓库到本地： git clone  https://github.com/ 用户名/原仓库名.git   
3. 创建功能分支开发： git checkout -b feature/new-feature   
4. 提交代码并推送到自己的Fork  
5. 在原仓库页面发起Pull Request，等待维护者审核合并
