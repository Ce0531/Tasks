# 1.CSS基础   
1. 简介
 - 全称：层叠样式表（Cascading Style Sheets）  
 - 作用：标记语言，用于为 HTML 结构设置样式，实现结构与样式的分离  
2. 编写位置  
CSS 有三种编写位置，各有适用场景和优缺点，核心区别在于样式的复用性和分离程度：

|样式类型|编写方式|优点|缺点|  
|---|---|---|---|  
|行内样式|写在HTML标签的style属性中，例：`<h1 style="color:red;font-size:20px;">标题</h1>`|精准控制单个标签样式|不能复用，违背结构与样式分离思想，属性值需紧跟 CSS 语法（名：值；）|  
|内部样式|所有 CSS 代码集中写在 HTML 的`<style>`标签中，一般放在`<head>`里|同一 HTML 文件内可复用样式|未实现结构与样式完全分离，仅支持单个 HTML 文件复用|  
|外部样式|写在独立的.css文件中，HTML 通过`<link>`引入，例：`<link rel="stylesheet" href="样式文件路径.css">`|实现结构与样式完全分离，多 HTML 文件可复用|需额外创建 CSS 文件，引入路径错误会导致样式失效|  
> 注意：<link>标签必须放在<head>标签中，用于说明引入文档与当前文档的关系  
3. 样式表优先级
 - 行内样式 > 内部样式 = 外部样式
 - 内部样式和外部样式优先级相同时，后面编写的样式会覆盖前面的
 - 同一样式表中，优先级与编写顺序相关，后面的样式覆盖前面的
4. CSS 语法规范
 - 基本结构：`选择器 {属性名 1: 属性值 1; 属性名 2: 属性值 2;}`  
 - 规范要求：选择器与大括号之间、属性名与属性值之间、不同属性声明之间，均需加一个空格  
 - 注释语法：`/* 注释内容 */`（快捷键：Ctrl+/）   
5. 代码书写风格  
 - 展开风格：推荐日常开发 / 维护使用，可读性高、便于调试  
   
   ```CSS
   div {
    width: 100px;
    height: 100px;
    background-color: red;
   }
   ```
 - 紧凑风格：推荐项目上线时使用，可减小文件体积
`div {width:100px;height:100px;background-color:red;}`  
# 2.CSS选择器  
选择器的核心作用：精准选择页面中的 HTML 元素，为其设置样式，分为基础选择器、复合选择器两大类    
1. 基础选择器  
最常用的基础选择器，是复合选择器的基础，各有专属使用场景：

|选择器类型|语法|作用|核心注意事项|  
|---|---|---|---|  
|通配符选择器|`* { 属性:值; }`|选择页面中所有 HTML 元素|全局样式初始化常用，例：`* {margin:0;padding:0;}`|   
|元素选择器|`标签名 { 属性:值; }`（例：`div { ... }`）|选择页面中某一种所有相同标签||
|元素选择器|`标签名 { 属性:值; }`（例：`div { ... }`）|选择页面中某一种所有相同标签|样式可复用，适用于批量设置同类型标签样式|  
|类选择器|`.类名 { 属性:值; }`，标签通过`class="类名"`绑定|选择页面中指定类名的任意元素|1. 类名不能以数字开头，多单词用短横线连接；2.一个元素可绑定多个类名（空格分隔）;3.多个元素可绑定同一个类名（核心复用特性）|  
|ID 选择器|`#ID值 { 属性:值; }`，标签通过`id="ID值"`绑定|精准选择页面中唯一的一个元素|1. ID 值尽量由字母、数字、短横线组成，不能以数字开头，无空格，区分大小写；2.一个元素只能有一个 ID，多个元素 ID 值不能重复;3.一个元素可同时拥有id和class属性|  
2. 复合选择器
由多个基础选择器组合而成，实现更精准的元素选择，核心为交集、并集选择器：
 - 交集选择器
   - 语法：`选择器1选择器2 { 属性:值; }`（例：`div.red { ... }、p#box { ... }`）
   - 作用：选择同时满足多个条件的元素
   - 注意  
    1. 有标签选择器时，标签名必须写在最前面  
    2. 不能出现两个及以上元素选择器（无意义）
 - 并集选择器
   - 语法：`选择器1,选择器2,选择器3 { 属性:值; }`（例：`div,p,.red { ... }`）
   - 作用：选择多个选择器对应的所有元素，实现集体声明
   - 注意：
     - 选择器之间用英文逗号分隔，逗号后可加空格  
     - 任意形式的基础选择器，都可作为并集选择器的一部分  
     - 可大幅缩小样式表体积，提升开发效率  
# 3.CSS 三大特性
CSS 的核心特性，决定了样式的生效规则，是解决样式冲突、实现样式复用的关键：
1. 层叠性
   - 核心：当同一元素的同一属性被设置多个样式时，会按优先级规则叠加 / 覆盖（优先级高的生效，优先级相同的后写覆盖先写）
   - 作用：解决样式冲突问题
 2. 继承性
   - 核心：子元素会自动继承父元素、祖先元素上设置的部分样式
   - 继承规则：就近原则（离元素越近的样式，优先级越高）
   - 可继承属性：文本相关属性（`font-*`、`text-*`、`color`等）、行高line-height等（与文字 / 文本相关的属性）
   - 不可继承属性：边框`border`、背景`background-*`、内外边距`margin/padding`、宽高`width/height`、定位相关属性等
3. 优先级  
    - 核心：不同选择器设置的样式，生效优先级不同，优先级高的样式覆盖优先级低的
    - 优先级从高到低：`!important` > 行内样式 > ID 选择器 > 类选择器 > 元素选择器 > 通配符选择器 > 继承的样式
>注意：`!important`需写在属性值后、分号前，例：`color:red !important;`，用于强制提升单个属性的优先级（慎用，避免破坏样式层级） 
# 4.CSS 颜色表示  
CSS 支持四种颜色表示方式，开发中常用RGB/RGBA和HEX/HEXA  
1. 英文单词  
  - 语法：`color: 颜色单词;`（例：`color:red;`、`color:blue;`）
  - 特点：简单直观，仅适用于基础颜色，无法表示复杂色值  
2. RGB/RGBA
  - 原理：通过红 (R)、绿 (G)、蓝 (B) 三原色的数值组合表示颜色，取值范围`0~255`；A 表示透明度，取值范围`0~1`（0 完全透明，1 完全不透明）
  - 语法：`rgb(红,绿,蓝) `/ `rgba(红,绿,蓝,透明度)`（例：`rgb(255,0,0)`、`rgba(255,0,0,0.5)`）
  - 注意：三原色数值需保持一致格式，要么都是整数，要么都是百分比
3. HEX/HEXA（十六进制）
  - 原理：将 RGB 三原色的数值转换为十六进制（0-9、a-f/A-F），6 位字符表示颜色，8 位字符（后两位）表示透明度
  - 语法：`#rrggbb` / `#rrggbbaa`（例：`#ff0000`、`#ff000080`）
  - 简写：当三原色的两位字符相同时，可简写为 3 位，例：`#ff0000` → `#f00`
 4. HSL/HSLA
  - 原理：通过色相 (H)、饱和度 (S)、亮度 (L) 表示颜色；A 表示透明度，取值范围`0~1`  
  - 色相：取值`0~360°`（0/360 为红色，120 为绿色，240 为蓝色）
  - 饱和度 / 亮度：取值`0%~100%`（饱和度 100% 为纯色，亮度 50% 为标准亮度）
  - 语法：`hsl(色相,饱和度,亮度) `/ `hsla(色相,饱和度,亮度,透明度)`（例：`hsl(0,100%,50%)`、`hsla(0,100%,50%,0.5)`）  
# 5.常用字体属性
字体属性用于设置文字的字体、大小、风格、粗细，支持单个属性设置和复合属性简写，所有字体属性均可继承。
1. 单个字体属性
     
  |属性名|作用|常用属性值|注意事项|  
  |---|---|---|---|  
  |`font-size`|设置字体大小|`px`（像素，最常用）、`em`、`rem`|需加单位，`body`中设置后，子元素可继承；浏览器默认字体大小为 16px|  
  |`font-family`|设置字体类型|具体字体名（例："微软雅黑"、"宋体"、Arial）；`serif`（衬线体）、`sans-serif`（非衬线体，推荐）|1. 多个字体用英文逗号分隔，按从左到右匹配（找到可用的即生效）；2. 中文字体名需加引号，西文字体名无需；3. 最后建议加通用字体（如sans-serif），避免字体缺失；4. Windows 系统默认字体为微软雅黑|  
  |`font-style`|设置字体风格|`normal`（正常，默认值）、`italic`（斜体，推荐，使用字体自身斜体）、`oblique`（倾斜，强制变形）|常用场景：取消`<em>`/`<i>`的默认斜体，例：`em {font-style:normal;}`|  
  |`font-weight`设置字体粗细|1. 关键字：`lighter`（更细）、`normal`（正常，默认）、`bold`（粗体）、`bolder`（更粗，多数字体不支持）；2. 数值：`100~900`（无单位，数值越大越粗）|数值对应：`100~300`≈`lighte`r、`400~500`≈`normal`、`600~900`≈`bold`；例：`font-weight:700;`（等效粗体）|  
 2. 字体复合属性（`font`）
   - 语法：`font: 字体风格 字体粗细 字体大小/行高 字体类型;`
   - 例：`font: italic bold 16px/24px "微软雅黑",Arial,sans-serif;`
   - 核心注意：
     1. 必须按顺序书写，不可随意调换  
     2. 字体大小和字体类型为必填项，其他为可选  
     3. 行高与字体大小之间用英文斜杠分隔，行高可省略单位（例：16px/1.5，行高为 16px×1.5）  
     4. 省略的属性会恢复为默认值（例：省略`font-style`，则为`normal`）  
# 6.常用文本属性
文本属性用于设置文字的颜色、间距、修饰、缩进、对齐、行高，是页面文字样式的核心，多数可继承。
1. 文本颜色（`color`）
  - 语法：`color: 颜色值;`
  - 支持：所有颜色表示方式（开发中常用 RGB/RGBA、HEX/HEXA）
  - 拓展：背景色`background-color`，用法与`color`一致，设置元素的背景颜色
 2. 文本间距  
    
  |属性名|作用|语法|注意事项|  
  |---|---|---|---|  
  |`letter-spacing`||设置字母 / 字符间距|`letter-spacing: 数值px;`|正值增大间距，负值减小间距，0 为默认|  
  |`word-spacing`|设置单词间距|`word-spacing: 数值px;`|仅对英文有效，以空格为单词分隔符；正值增大，负值减小|  
3. 文本修饰（`text-decoration`）
  - 语法：`text-decoration: 修饰样式 颜色;`（颜色可选）
  - 常用属性值  
     - `none`：无修饰线（最常用，取消`<a>`标签默认下划线，例：`a {text-decoration:none;}`）
     - `underline`：下划线（常用，例：`text-decoration:underline red;`）
     - `overline`：上划线
     - `line-through`：删除线（例：`<del>`标签默认样式）
     - `dashed`：虚线修饰线  
     - `wavy`：波浪线修饰线   
4. 文本缩进（`text-indent`）
  - 语法：`text-indent: 数值px;`（或`em`，推荐：1em = 当前字体大小，实现 “首行缩进 2 字符”：`text-indent:2em;`）
  - 作用：设置段落首行缩进，仅对块元素生效
5. 文本水平对齐（`text-align`）
  - 语法：`text-align: 属性值;`
  - 常用属性值：`left`（左对齐，默认）、`right`（右对齐）、`center`（居中对齐）
  - 作用：设置块元素内部的文本、行内元素、行内块元素的水平对齐方式（对块元素自身无效）
6. 行高（`line-height`）
  - 语法：`line-height: 数值px / 纯数字 / 百分比;`
  - 常用属性值  
     - `normal`：浏览器默认值（约为字体大小的 1.2 倍）
     - 固定值：`24px`（最常用，精准控制）
     - 纯数字：`1.5`（行高为当前字体大小的 1.5 倍，推荐，子元素可继承比例）
     - 百分比：`150%`（行高为当前字体大小的 150%）
  - 核心注意：
     1. 行高过小会导致文字重叠，属性值不能为负数  
     2. `line-height`可继承，为便于控制，优先使用纯数字  
     3. 与`height`的关系：1. 未设置`height`，元素高度由`line-height`计算得出；2. 设置`height`，元素高度为`height`的值；
     4. 应用场景：1. 控制多行文字的行间距；2. 单行文字垂直居中：设置元素`height = line-height`
7. 文本垂直对齐（`vertical-align`）
 - 作用：指定同一行的行内元素 / 行内块元素之间，或表格单元格内的文本垂直对齐方式
 - 常用属性值：`baseline`（基线对齐，默认）、`top`（顶部对齐）、`middle`（中部对齐）、`bottom`（底部对齐）  
 - 注意：不能控制块元素的垂直对齐；解决行内块元素（如图片）与文字的对齐问题，常用`vertical-align:middle;`  
# 7.列表相关属性
列表属性仅作用于有序列表`<ol>`和无序列表`<ul>`，用于设置列表标记的样式、位置、自定义图标，支持单个属性和复合属性
1. 单个列表属性  

|属性名|功能|常用属性值|  
|---|---|---|  
|`list-style-type`|设置列表标记样式|`none`（无标记，最常用，用于取消默认列表符号）、`square`（实心方块）、`disc`（实心圆点，默认）、`decimal`（阿拉伯数字）、`lower-roman`（小写罗马数字）、`upper-roman`（大写罗马数字）、`lower-alpha`（小写字母）、`upper-alpha`（大写字母）|  
|`list-style-position`|设置列表标记位置|`outside`（标记在`<li>`外部，默认）、`inside`（标记在`<li>`内部）|  
|`list-style-image`|自定义列表标记为图片|`url(图片地址)`（例：`list-style-image:url(icon.png);）`|  
2. 列表复合属性（`list-style`）
  - 语法：`list-style: 标记样式 标记位置 自定义图片;`
 - 例：`list-style: none outside url(icon.png);`
 - 注意：无数量和顺序要求，省略的属性恢复默认值；开发中最常用：`list-style:none;`（取消默认列表标记）  
# 8.表格相关属性  
表格属性分为通用边框属性（所有元素可用）和表格独有属性（仅表格 / 单元格可用），用于美化表格样式，实现边框合并、列宽固定等效果
1. 通用边框属性
所有 HTML 元素均可使用，用于设置元素的边框宽度、颜色、样式，支持单个属性和复合属性   

|属性名|功能|常用属性值|  
|---|---|---|  
|`border-width`|设置边框宽度|CSS长度值|   
|`border-color`|设置边框颜色|所有 CSS 颜色值|  
|`border-style`|设置边框样式（必填，否则边框不显示）|`none`（无边框，默认）、`solid`（实线，最常用）、`dashed`（虚线）、`dotted`（点线）、`double`（双实线）|  
|`border`|边框复合属性|`宽度 样式 颜色`，无顺序要求|  
2. 表格独有属性
仅`<table>`标签可用，是表格样式的核心，实现边框合并、列宽固定等关键效果  

|属性名|功能|常用属性值|注意事项|  
|---|---|---|---|  
|`table-layout`|设置列宽计算方式|`auto`（自动，列宽根据内容计算，默认）、`fixed`（固定，列宽平均分配）|`fixed`需配合`<table>`的`width`使用，提升表格渲染效率|  
|`border-spacing`|设置单元格之间的间距|CSS 长度值|生效前提：`border-collapse:separate;`（边框不合并）|  
|`border-collapse`|设置单元格边框是否合并|`collapse`（合并，最常用，例：`border-collapse:collapse;`）、`separate`（不合并，默认）|合并后，`border-spacing`失效|  
|`empty-cells`|设置无内容单元格是否显示边框|`show`（显示，默认）、`hide`（隐藏）|生效前提：`border-collapse:separate;`（边框不合并）|  
|`caption-side`|设置表格标题`<caption>`的位置|`top`（表格上方，默认）、`bottom`（表格下方）|仅控制`<caption>`的位置|  
# 9.背景相关属性  
背景属性用于设置元素的背景颜色、背景图片、平铺方式、位置，支持单个属性和复合属性，不可继承，所有元素均可使用。
1. 单个背景属性   
   
|属性名|功能|常用属性值 / 语法|  | 
|---|---|---|---|  
|`background-color`|设置背景颜色|所有 CSS 颜色值，默认`transparent`（透明）|  
|`background-image`|设置背景图片|`url(图片地址)`，无图片则设为none|  
|`background-repeat`|设置背景图片平铺方式|`repeat`（水平 + 垂直平铺，默认）、`repeat-x`（仅水平平铺）、`repeat-y`（仅垂直平铺）、`no-repeat`（不平铺，最常用）|  
|`background-position`|设置背景图片显示位置|1. 关键字：水平（`left`/`center`/`right`）、垂直（`top`/`center`/`bottom`）；2. 数值：`xpx ypx`（以元素左上角为原点，x 为水平偏移，y 为垂直偏移）；3. 百分比：`x% y%`,例：`50% 50%`（居中）|仅设置一个值，另一个值默认`center`（例：`left` →` left center`）|  
2. 背景复合属性（`background`）
 - 语法：`background: 颜色 图片 平铺 位置;`
 - 例：`background: #f5f5f5 url(bg.png) no-repeat center center;`
 - 注意：无数量和顺序要求，省略的属性恢复默认值；开发中常用简写，提升效率  
# 10.鼠标相关属性（`cursor`）
 - 作用：设置鼠标悬浮在元素上时的光标样式
 - 语法：`cursor: 属性值;`
 - 常用属性值：
   - `pointer`：小手（最常用，设置可点击元素，如`<a>`/ 按钮）
   - `move`：移动光标（表示元素可拖拽）
   - `text`：文本光标（表示可编辑文本）
   - `crosshair`：十字线光标
   - `wait`：等待光标（表示页面 / 元素正在加载）
   - `help`：帮助光标（表示有帮助信息）
>拓展：自定义光标，例：`cursor: url(光标图片地址), pointer;`（自定义图片失效时，显示默认小手）  
# 11.CSS 盒子模型
1. 核心概念  
CSS 将所有 HTML 元素视为一个矩形盒子，盒子由外到内分为四层：外边距（margin） → 边框（border）→ 内边距（padding） → 内容区（content），是布局的核心基础
2. 常用长度单位
 - `px`：像素，绝对单位，固定大小，最常用（开发中优先使用）
 - `em`：相对单位，1em = 当前元素的`font-size`（若未设置，继承父元素的`font-size`）
 - rem：相对单位，1rem = 根元素`<html>`的`font-size`（推荐做响应式布局）  
> 注意：CSS 中设置长度必须加单位（除行高纯数字、百分比外）
3. 盒子模型各部分属性
 - 内容区（content）
控制盒子的核心宽高，仅作用于块元素 / 行内块元素，行内元素宽高由内容决定，不可手动设置  

|属性名|功能|注意事项|  
|---|---|---|  
|`width`|设置内容区宽度	|不可为负数|  
|`height`|设置内容区高度|不可为负数，未设置时，块元素高度由内容决定|  
|`max-width`|设置最大宽度|防止元素宽度过大，超出容器，不与`width`同时使用|  
|`min-width`|设置最小宽度|防止元素宽度过小，内容挤压，不与`width`同时使用|  
|`max-height`|设置最大高度|超出部分可配合`overflow`处理|  
|`min-height`|设置最小高度|防止元素高度过小，内容挤压|   
 - 内边距（padding）  
内容区与边框之间的距离，不可为负数，会增大元素的实际占用宽度 / 高度（未设置`box-sizing:border-box`时） 

|属性名|功能|复合属性语法（`padding`）|  
|---|---|---|  
|`padding-top`|上内边距|1.` padding: 数值;` → 四个方向内边距相同；2. `padding: 上 左右;` → 两个值；3.` padding: 上 左右 下;` → 三个值；4. `padding: 上 右 下 左;` → 四个值（顺时针）|  
|`padding-right`|右内边距||   
|`padding-bottom`|下内边距	||  
|`padding-left`|左内边距||  
> 注意：行内元素的上下内边距无法完美设置（视觉上有效果，但不占用实际空间），左右内边距可正常设置；块元素 / 行内块元素四个方向内边距均可完美设置  
 - 边框（border）
见通用边框属性，边框会增大元素的实际占用宽度 / 高度（未设置`box-sizing:border-box`时）  
 - 外边距（margin）
盒子与其他盒子之间的距离，可为负数 /auto，不会增大元素自身的宽高，仅控制元素的位置  

|属性名|功能|复合属性语法（`margin`）|
|---|---|---|  
|`margin-top`|上外边距|与`padding`复合属性语法完全一致（1/2/3/4 个值，顺时针）|  
|`margin-right`|右外边距	||  
|`margin-bottom`|下外边距||   
|`margin-left`|左外边距||  
> 注意:
> 1. 外边距的参考系：以父元素 / 相邻元素的 content 区为基准
> 2. 外边距的影响：上 / 左外边距影响自身位置，下 / 右外边距影响后续元素位置
> 3. 元素类型限制： 块元素 / 行内块元素：四个方向外边距均可正常设置；行内元素：上下外边距无效，左右外边距可正常设置
> 4. `margin:auto`：仅对块元素生效，设置`margin-left: auto; margin-right: auto;`（简写：`margin:0 auto;`），可实现块元素在父元素中水平居中（需设置块元素的宽度，否则宽度为 100%，居中无意义）
> 5. 外边距可为负数：用于调整元素的偏移位置，实现特殊布局  
4. 盒子模型关键问题
 - margin 塌陷（父子元素）
   - 问题：子元素的上外边距会作用到父元素上，导致父元素一起向下偏移；子元素的下外边距会作用到父元素的后续元素上。
   - 解决方法：
    1. 给父元素设置上边框
    2. 给父元素设置上内边距
   3. 给父元素设置 CSS 样式`overflow:hidden;`（最常用，无视觉影响）
 - margin 合并（兄弟元素）
   - 问题：上下相邻的两个兄弟块元素，上方元素的下外边距和下方元素的上外边距会合并，取最大值生效，而非相加
   - 解决方法：布局时，仅给一个兄弟元素设置上下外边距（例：只给下方元素设置上外边距），避免合并  
5. 元素显示模式  
元素的显示模式决定了元素的布局特性（宽高设置、同行排列、内外边距生效等），分为块元素、行内元素、行内块元素，可通过`display`属性修改  
 - 三大显示模式特性
    
  |显示模式|代表元素|核心特性|  
  |---|---|---|  
  |块元素（block）|`div`、`p`、`h1-h6`、`ul/ol/li`、`header`等|1. 独占一行；2. 可手动设置宽高 / 内外边距；3. 未设置宽度时，宽度为父元素 content 区的 100%；4. 可容纳块元素 / 行内元素 / 行内块元素（`<p>`/`<h1-h6>`不能容纳块元素）|  
  |行内元素（inline）|`span`、`a`、`em`、`i`、`strong`等|1. 可与其他行内元素 / 行内块元素同行排列；2. 不可手动设置宽高，宽高由内容决定；3. 上下内外边距无效，左右内外边距有效；4. 仅可容纳行内元素 / 文本（`<a>`可容纳块元素，需修改显示模式）|  
  |行内块元素（inline-block）|`img`、`input`、`button`|1. 可与其他行内元素 / 行内块元素同行排列；2. 可手动设置宽高 / 内外边距；3. 既有行内元素的同行特性，又有块元素的样式设置特性|   
  - 修改显示模式（`display`）
    - 语法：`display: 属性值;`
    - 常用属性值：
    `none`：隐藏元素，不占用任何页面空间（与`visibility:hidden`区别：后者隐藏但保留元素空间）
    `block`：将元素转为块元素
   `inline`：将元素转为行内元素
   `inline-block`：将元素转为行内块元素  
6. 溢出处理（`overflow`）
 - 作用：处理元素内容区超出元素宽高的部分（例：文字过多、图片过大）  
 - 语法：`overflow: 属性值;`
 - 常用属性值：
   - `visible`：溢出部分正常显示（默认）
   - `hidden`：溢出部分隐藏（最常用，避免内容挤压布局）
   - `scroll`：无论是否溢出，都显示滚动条（水平 + 垂直）
   - `auto`：自动判断，溢出时显示滚动条，不溢出时不显示（推荐）   
# 12. 元素默认样式与布局技巧  
1.  元素默认样式  
浏览器会为部分 HTML 元素设置默认样式，导致布局出现意外间距，开发中需先重置默认样式  
常见元素默认样式：
 - `<a>`：默认下划线、字体蓝色、鼠标小手pointer；
 - `<h1-h6>`：默认文字加粗、字体大小递减、上下外边距；
 - `<p>`：默认上下外边距；
 - `<ul>/<ol>`：默认左内边距、默认列表标记；
 - `<body>`：默认 8px 的四个方向外边距；
 - `<img>`：默认行内块，与文字基线对齐，有默认间距。
> 注意：元素的默认样式 > 继承的样式，重置默认样式时，选择器需直接选择该元素   
2. 核心布局技巧
 - 行内 / 行内块元素的文本式处理:行内元素、行内块元素，可被父元素当做文本处理，直接使用文本属性控制其布局  
   - 水平对齐：父元素设置`text-align:center/left/right;`
   - 垂直居中：父元素设置`height = line-height + 子元素设置vertical-align:middle;`
   - 首行缩进：父元素设置`text-indent;`（仅对第一个子元素生效）  
 - 子元素在父元素中水平居中  
  
  |子元素显示模式|实现方法|注意事项|  
  |---|---|---|  
  |块元素|子元素设置：`width: 固定值;margin:0 auto;`|必须设置子元素固定宽度，否则宽度 100%，居中无意义|  
  |行内 / 行内块元素|父元素设置：`text-align:center;`|无需设置子元素宽高，直接继承父元素的文本对齐特性|  
  - 子元素在父元素中垂直居中  

  |子元素显示模式|实现方法|注意事项|  
  |---|---|---|   
  |块元素|子元素设置：`margin-top: (父元素content高度 - 子元素盒子总高) / 2;`|需知道父元素和子元素的精确高度，适用于固定高度布局|  
  |行内 / 行内块元素|1. 父元素设置：`height = line-height;`2. 子元素设置：`vertical-align:middle;`|1. 若想绝对垂直居中，可将父元素`font-size:0;`2. 无高度限制，适用于动态内容布局（最常用）|   
 - 行内 / 行内块元素的同行间距问题  
   - 问题：行内元素 / 行内块元素同行排列时，标签之间的换行 / 空格会被浏览器解析为一个空格，导致元素之间出现默认间距  
   - 解决方法：
     1. 去掉 HTML 标签之间的换行和空格（直接写在同一行）
     2. 给父元素设置`font-size:0;`，再给需要显示文字的子元素单独设置`font-size`（最常用，无视觉影响）
     3. 将行内块元素转为块元素，通过浮动 / 定位布局  
 - 行内块元素的垂直对齐问题  
   - 问题：行内块元素（如`<img>`）与文字同行时，默认基线对齐，导致视觉上不对齐  
   - 解决方法：
     1. 给行内块元素设置`vertical-align:middle/top/bottom;（`最常用，推荐middle）  
     2. 若父元素中只有一张图片，将图片设置为`display:block;`（转为块元素，取消基线对齐）  
     3. 给父元素设置`font-size:0;`，若内部有文字，单独给文字设置`font-size`
