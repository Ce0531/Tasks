# 1️⃣概述  
## 1. 关于HTML    
  - HTML 是超文本标记语言，并非编程语言
  - 可设置默认浏览器，后续文档用该浏览器打开
  - 写源代码，交给浏览器进行渲染  
## 2. 标签（元素）    
 - 标签分为双标签（有开始和结束标签）和单标签（自闭合标签）
 - 标签间存在并列关系、嵌套关系，可使用 Tab 键进行缩进，保证代码整洁
 - 书写规范：标签必须正确闭合，嵌套标签不可交叉   
## 3. 标签属性    
 - 部分标签有自带的默认属性，属性需写在开始标签内，格式为属性名="属性值"  
 - 禁用属性示例： ```
  <input disabled>```（disabled 为布尔属性，可直接写）  
 - 一个标签中不能出现重复属性，否则会失效,错误示例：``` <input type="text" type="password">```   
## 4. HTML基本结构    
```
<html>
  <head>
    <!-- 头部信息，如编码、标题等 -->
  </head>
  <body>
    <!-- 页面主体内容，显示在浏览器窗口 -->
  </body>
</html>  
 ```  
## 5. 注释  
 - 语法：``` <!-- 注释内容 -->```  
 - 注意事项：注释不能嵌套  
 - 快捷键：ctrl + /（快速添加 / 取消单行注释）  
## 6. 文档声明  
 - 语法：``` <!DOCTYPE html>```（声明当前文档为 HTML5 文档）  
 - 注意事项：必须写在页面第一行，位于```  <html>``` 标签之前，无结束标签  
## 7. 字符编码  
 - 语法：``` <meta charset="UTF-8"/>```（单标签，定义文档的字符编码）
 - 书写位置：放在``` <head>``` 标签内部
 - 作用：UTF-8 是通用字符编码，可避免页面出现乱码   
## 8. 设置页面语言  
 - 语法：```<html lang="zh-CN">```
 - 作用：定义当前页面的语言为中文，利于搜索引擎识别、浏览器解析和无障碍设备读取     
## 9. HTML 标准结构  
 ```
   <!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8"/>
    <title>标题</title> <!-- 页面标题，显示在浏览器标签栏 -->
  </head>
  <body>
    <!-- 页面主体内容 -->
  </body>
</html>  
```

MDN 官方文档：developer.mozilla.org，是 HTML/CSS/JS 最权威的参考文档  
# 2️⃣语法学习  
## 1.排版标签  
 |标签|作用|  
 |---|---|  
 |```<h1>~<h6>```|标题|  
 |```<p>```|段落|  
 |```<div>```|无任何含义，用于整体布局|  
 - ```<h1>```最好只写一个，```<h2~h6>```可适量  
 - ```<h1~h6>```不能互相嵌套  
 - ```<p>```不能嵌套```<h1~h6>```,```<p>```,```<div>```  
## 2.语义化标签 
 - 核心原则:标签的默认样式不重要，语义优先（即根据内容的含义选择对应的标签，而非根据样式）  
 - 优势:代码结构清晰，可读性强，便于开发和维护;有利于 SEO（搜索引擎优化），搜索引擎更易识别页面内容;方便无障碍设备（如盲人阅读器）解析页面内容  
## 3.块级元素与行内元素
 1. 块级元素
 - 特点：独占一行   
 - 常见示例：div、h1~h6、p 等  
 2. 行内元素
 - 特点：不独占一行  
 - 常见示例：span、a、em、strong、img  
 3. 嵌套规则
 - 块级元素中可嵌套行内元素和其他块级元素（部分特殊块级元素除外）
 - 行内元素中只能嵌套行内元素，不可嵌套块级元素
 - 特殊规则：```<p>``` 标签中不要嵌套块级元素（如 div、h1 等）  
## 4.文本标签（通常为行内元素）
 1. em：强调需要着重阅读的内容（斜体）
 2. strong：标记十分重要的内容（加粗）
 3. span：无语义，包裹短语的通用容器
 4. 不常用  
  - 代码相关：```<code>```（代码）、```<samp>```（输出）、```<kbd>```（键盘）、```<var>```（变量）
  - 格式相关：```<sub>/<sup>```（上下标）、```<bdo>```（方向）、```<del>/<ins>```（增删）
  - 引用相关：```<q>```（短引用）、```<blockquote>```（长引用）、```<cite>```（作品名）
  - 语义辅助：```<abbr>```（缩写）、```<dfn>```（术语）、```<address>```（地址）、```<small>```（细则）  
## 5.图片标签
1. 基本语法  
```<img width="" height="高度" src="图片路径" alt="图片描述">```  
2. 注意:默认情况下 img 为行内元素  
3. alt 属性的作用：
 - 搜索引擎可识别图片的内容  
 - 当图片无法展示时，部分浏览器会显示 alt 的值  
 - 盲人阅读器会朗读 alt 的值  
## 6.相对路径和绝对路径  
1. 相对  
 -  ```../```上一级  
 -  ```./```当前位置（相对路径中不可用）  
 -  ```/```下一级  
2. 绝对  
本地绝对路径，网络绝对路径  
## 7.常见图片格式  
1. jpg/jpeg 格式  
 - 类型：有损压缩格式  
特点：色彩丰富、占用空间小，不支持透明、动态图  
 - 使用场景：对图片细节无极高要求的外网传输使用  
2. png 格式  
 - 类型：无损压缩格式  
 - 特点：色彩丰富，占用空间稍大，支持透明背景，不支持动态图  
 - 使用场景：图片有透明背景、需要更高质量呈现（如公司 logo）  
3. bmp 格式  
 - 类型：不进行压缩  
 - 特点：色彩丰富，占用空间极大，不支持透明背景、动态图  
 - 使用场景：对图片精度要求极高的场景（如大型游戏中），网页中很少使用  
4. gif 格式  
 - 类型：仅支持 256 种颜色，色彩呈现不够完善  
 - 特点：支持的颜色数少，支持简单透明效果，支持动态图  
 - 使用场景：网页中的动态图  
5. webp 格式  
 - 特点：兼具 jpg 和 png 的优点，但兼容性一般  
 - 使用场景：网页中的各种图片    
6. base64 格式  
 - 本质：一种文本编码格式，需要通过浏览器打开，其他应用通常无法直接打开  
 - 使用场景：一些较小的图片，或需要和页面一起加载的图片    
## 8.超链接标签```<a>```
1. 跳转页面  
 - 语法：```<a href="页面路径"></a>```  
2. 跳转文件  
 - 直接打开：```<a href="文件路径"></a>```  
 - 自动触发下载（如 zip 文件）：文件为压缩包等格式时，浏览器会自动触发下载
 - 强制触发下载：```<a href="文件路径" download="文件名"></a>```
3. 跳转锚点
 - 设置锚点（两种方法）
   - 方法 1：a 标签设置 name 属性 ```<a name="tt1"></a>```  
   - 方法 2：其他标签设置 id 属性 ```<h2 id="t2">位置</h2>```  
 - 注：带 id 属性的锚点优先级高于 name 属性；name 属性区分大小写，锚点跳转建议使用 id  
 - 跳转锚点的写法  
   - 跳转到指定锚点：`<a href="#tt1">跳转到锚点</a>`
   - 回到页面顶部：`<a href="#">回到顶部</a>`  
   - 刷新本页面：`<a href="">刷新本页面</a>`  
   - 执行 js 代码：`<a href="javascript:alert(1);">点我弹窗</a>`  
4. 唤起指定应用
 - 拨号：`<a href="tel:电话号码">电话联系</a>`  
 - 发送邮件：`<a href="mailto:邮箱地址">邮件联系</a>`  
 - 发送短信：`<a href="sms:电话号码">短信联系</a>`

## 10.表格标签
1. 基本结构
表格由表格标题、表格头部、表格主体、表格脚注组成  
2. 核心标签  
 - `<table>`	表格容器	
 - `<caption>`	表格标题	
 - `<thead>`	表格头部	
 - `<tbody>	`表格主体	
 - `<tfoot>`	表格脚注	
 - `<tr>`	表格的行	
 - `<th>`	表头单元格	
 - `<td>`	普通单元格  
3. 基础语法  
```
<table>
  <caption>表格标题</caption>
  <thead>
    <tr>
      <th>表头1</th>
      <th>表头2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>数据1</td>
      <td>数据2</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>脚注1</td>
      <td>脚注2</td>
    </tr>
  </tfoot>
</table>  
```  
4. 常用属性
 - 表格`<table>`属性
   - width：设置表格宽度
height：设置表格最小高度，表格最终高度可能比设置的值大
   - border：设置表格边框宽度（仅控制最外侧边框，单元格边框需后续 CSS 控制）
 - 单元格`<th>/<td>`属性
   - width：设置单元格的宽度，同列所有单元格全都受影响
   - height：设置单元格的高度，同行所有单元格全都受影响
   - align：设置单元格的水平对齐方式
   - valign：设置单元格的垂直对齐方式（可选值：top（顶部）、middle（居中）、bottom（底部））
   - rowspan：指定单元格要跨的行数
   - colspan：指定单元格要跨的列数
5. 表格通用注意点
 - table 元素的 border 属性仅能控制表格最外侧边框的宽度，单元格边框需用 CSS 控制  
 - 给某个单元格设置宽度后，它所在列的宽度就确定  
6. 表格配套单标签  
 - 换行：`<br>`（单标签），放在单元格内实现段落内换行，一般用于大段文本中  
 - 分割线：`<hr>`（单标签），用于分隔表格内容  
 - 注意：不要用`<br>`增加文本之间的间距，用 CSS 实现；`<hr>`的语义是分隔，单纯想加横线建议用 CSS	  
## 11.表单标签
1. 表单基本结构  
 - 核心属性  
   - action：用于指定表单的提交地址  
   - target：用于指定表单提交后页面的打开方式，常用值：_self（当前页面）、_blank（新页面）
   - method：用于控制表单的提交方式，常用值：get、post  
 - 基础语法  
 ```
 <form action="提交地址" method="post" target="_self">
  <!-- 表单控件 -->
  <input type="text" name="username">
  <button>提交</button>
 </form>
 ```  
2. 常用表单控件  
 - 文本输入框  
 `<input type="text" name="自定义名称" value="默认值" maxlength="最大输入长度">`   
   - name：提交数据的名称（表单数据提交必备）  
   - value：输入框的默认输入值  
   - maxlength：最大输入长度  
 - 密码输入框
`<input type="password" name="自定义名称">`
   - 常用属性同文本输入框，value 一般不用（无实际意义）  
 - 单选框
 ```
 <input type="radio" name="sex" value="male"> 男
 <input type="radio" name="sex" value="female"> 女
 ```  
   - name 属性值必须相同，才能实现单选互斥
   - value：单选框提交的实际数据  
 - 隐藏域
`<input type="hidden" name="自定义名称" value="固定数值">`
   - 作用：用户不可见的输入框，提交表单时传递一些固定数据  
 - 按钮
   - button 标签默认 type 为 submit（提交表单）：`<button>提交</button>`
   - 显式设置提交按钮：`<button type="submit">提交</button>`
   - 普通按钮（不提交表单）：`<button type="button">普通按钮</button>`
   - 注意：button 标签一般不要指定 name 属性  
 - 下拉框
   - 语法：
```
<select name="自定义名称">
  <option>选项1</option>
  <option value="option2">选项2</option>
</select>
```
   - 若 option 未设置 value 属性，提交的是 option 标签中间的文本内容
   - 若设置 value，提交的是 value 的属性值
3. 表单通用属性
 - 禁用属性：disabled
 - 适用标签：input、textarea、button、select、option
 - 作用：禁用表单控件，禁用后控件无法点击、无法提交  
4. 表单辅助标签  
 - label 标签
   - 作用：可与表单控件关联，关联后点击 label，对应的表单控件会获焦
   - 语法示例：`<label for="input1">用户名</label><input type="text" id="input1" name="username">`  
 - fieldset & legend
   - 作用：表单控件分组 + 分组标题
语法：
```
<fieldset>
  <legend>用户信息</legend>
  <!-- 分组内的表单控件 -->
  <input type="text" name="username">
</fieldset>
```
 - 文件上传标签
   - 语法：`<input type="file" name="file">`
5. 框架标签：iframe
 - 语法：`<iframe src="嵌入页面路径" border="0"></iframe>`
 - border 属性：设置框架边框，值为 0 则无边框
 - 作用：在当前页面中嵌入另一个页面（如广告、其他网页内容）
6. 超链接 & 表单通用属性  
 - target 属性：均可通过 target 控制页面 / 文件的打开方式

## 13.HTML 全局属性
全局属性可用于大部分 HTML 标签，核心全局属性如下：
|属性名|含义说明|  
|---|---|
|id|标签唯一标识，值不可重复|
|class|	标签类名，可重复，多个标签可共用一个 class|
|style|	给标签设置行内 CSS 样式|
|dir|	控制内容的排列方向，可选 ltr（从左到右）、rtl（从右到左）|
|title|	给标签设置文字提示，鼠标悬浮时显示|
|lang|	给标签设置语言类型|	
## 14.meta元信息   
1. 配置字符编码
`<meta charset="utf-8">`
2. 针对 IE 浏览器的兼容性配置
`<meta http-equiv="X-UA-Compatible" content="IE=edge">`
3. 针对移动端的配置（移动端课程中会详细讲解）
`<meta name="viewport" content="width=device-width, initial-scale=1.0">`
4. 配置网页关键字
`<meta name="keywords" content="8-12个以英文逗号分隔的单词/词语">`
5. 配置网页描述信息
`<meta name="description" content="80字以内的一段话，与网站内容相关">`
6. 针对搜索引擎爬虫配置
`<meta name="robots" content="此处可选值见下表">`
配置值与描述对照表  
| 值 | 描述 |
| --- | --- |
| index | 允许搜索引擎爬虫索引当前页面 |
| noindex | 禁止搜索引擎爬虫索引当前页面 |
| follow | 允许爬虫跟随页面中的链接继续抓取 |
| nofollow | 禁止爬虫跟随页面中的链接继续抓取 |
| all | 等价于 `index,follow`，允许索引和跟随链接 |
| none | 等价于 `noindex,nofollow`，禁止索引和跟随链接 |
| noarchive | 禁止搜索引擎缓存页面快照 |
| nosnippet | 禁止搜索引擎在搜索结果中显示页面摘要 |
| noodp | 禁止使用Open Directory项目中的描述作为搜索结果摘要 |
| noydir | 禁止使用Yahoo目录中的描述作为搜索结果摘要 |
7. 配置网页作者
`<meta name="author" content="tony">`  
8. 配置网页生成工具
`<meta name="generator" content="Visual Studio Code">`  
9. 配置定义网页版权信息
`<meta name="copyright" content="2023-2027©版权所有">`
10.  配置网页自动刷新
`<meta http-equiv="refresh" content="10;url=http://www.baidu.com">`
