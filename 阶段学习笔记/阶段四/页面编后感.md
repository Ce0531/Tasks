# 页面编后感  
### 一.构想  
在复刻前我认真看了预览效果图和呈现视频，我的预期是尽量把网页全部复刻出来，所有的功能与交互都尽量实现，并在时间充足的情况下，尝试移动端适配的问题。最后的成品差不多达到了我的预期，不过还有些小问题我询问了ai也没有解决。  
### 二.HTML,CSS,JS的运用  
#### 1.HTML：搭建网页骨架  
用 HTML 搭建网站的 “骨架” 时，我按照页面模块划分结构，从导航栏、主界面、Introduction、招新要求、Q&A 板块到联系方式，用语义化标签梳理出清晰的页面层级，确保每个模块的结构合理、逻辑通顺.  
#### 2.CSS：美化网页  
我先会根据VSCode给的属性，然后加上我已知需要的属性，设定一个自认为合适的属性值，然后点开网页看效果，再根据所给的网页预览效果图一点点改属性值，如果有其他细节缺失会去询问AI，然后下一次出现同样效果就可以知道要用什么属性。我先做好页面排版与静态美化，当效果与预览图差不多时再去添加动态效果。  
#### 3.JS: 实现交互  
由于第三阶段js学的比较少，所以复刻网页时并不知道哪些地方需要用到js，当做到qa部分时发现自己怎么也无法用css实现完美下拉效果，询问AI后用js解决了，不过中间仍存在问题。最后想实现回到顶部的小按钮，因为js不太熟练，所以选择了用css来做。  
### 三.最大难题与解决  
 - 问题：Q&A手风琴折叠功能的异常问题。最初点击问题时，回答区域只会轻微下移，无法完整展开，多次修改CSS和JS后仍未解决。通过浏览器开发者工具排查，我找到了问题的根源：一是 CSS 过渡属性不匹配，用`transition: transform`却修改`height`，导致动画完全失效；二是`height: auto`不支持CSS过渡，无法实现从0到内容高度的平滑展开；三是JS与CSS逻辑冲突JS控制`display`的同时CSS控制`max-height`，两套规则互相干扰。   
 - 解决：重构了交互逻辑，让JS仅负责切换`active`类名,CSS全权控制`max-height`和`padding`的过渡动画，最终实现了效果，也彻底解决了内容显示不全的问题。除此之外，我还遇到了容器布局调整、样式优先级覆盖、DOM 加载顺序导致 JS 失效等问题,在AI帮助下解决
