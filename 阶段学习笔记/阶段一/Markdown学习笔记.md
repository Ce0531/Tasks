# 1️⃣语法
### 1. 标题分级    
- 一级标题： # 标题内容 （一个#加空格，字号最大）   
- 二级标题： ## 标题内容 （两个#加空格）  
- 三级标题： ### 标题内容 （三个#加空格）  
以此类推，最多六级标题，#和文字之间必须加空格，不然不生效  
### 2. 无序列表    
规则：每一行开头用  - 空格     
正确效果：每一行都有黑点，文字完全对齐
### 3. 有序列表  
规则：每一行用  数字. 空格  开头，数字后必须加空格，内容直接写，不用额外缩进
### 4. 任务列表  
规则:' - [ ] ' [^02]
[^02]:为了可以展示正确用法，故用‘’括起来  
### 5. 换行/分段  
- 单纯换行（行与行紧挨着）：行尾敲2个空格，再按回车  
- 分段（行与行有空隙）：直接空一行，再写下一段内容  
### 6. 文本简单格式  
- 加粗： **文字内容** （两个星号包裹文字）  
- 斜体： *文字内容* （一个星号包裹文字）
- 粗斜体：***文字内容***（三个星号包裹文字）
- 删除线：~~文字内容~~（两个~~包裹文字）
### 7.链接
- [链接名字](链接地址)
- 拓展：锚点链接（适用于刚开始不知道网址或出现多次同一网址）
  [文字标题][锚点名称]     文章最后输入[锚点名称]：网址   
### 8.引用  
- 单行：>+空格
- 嵌套：多个>，每多一个，更深一层
### 9.脚注  
  文章内容[^序号]   
  [^序号]：脚注内容  
### 10.图片  
- 规则:![替代文本]（图片路径“可选图片标题”）
- 带链接的图片：将图片的Markdown语法作为链接的“链接文本”部分
### 11.表格  
- 表头：用|来分隔
- 分隔行：用|来分隔，并使用至少三个-来创建
- 左对齐: :---  
  右对齐：---:     
  居中对齐：：---：
- eg:
  | 歌手 |性别|  
  | ---  |---|  
  | 陶喆 |男  |    
### 12.代码块  
- 行内代码：前后各用`来包裹（1前面的按键）
- 代码块：上下各用```来包裹，第一行三个`后面可以指定代码类型    
### 13.注意  
- 标题和列表之类的一定记得带空格
- 换行记得在文末打两个空格  
# 2️⃣有关渲染  
### 1.核心执行主体  
Markdown 本身是纯文本标记语法，无法被浏览器/系统直接识别，渲染依赖解析器（Parser），不同场景下的执行主体不同：  
|场景|执行主体|技术实现|  
|---|---|---|  
|本地编辑器（VS Code、Typora）|编辑器内置解析器|前端 JS 库（marked、markdown-it）或原生解析逻辑|  
|网页端（GitHub、掘金、公众号）|服务端/客户端解析器|服务端 Node.js 解析（如 markdown-it）后返回 HTML，或前端 JS 渲染|  
|静态网站生成器（Hugo、Docusaurus）|构建时解析器|构建阶段将 .md 转为 HTML 静态文件|  
|桌面端工具（Obsidian）|应用内解析引擎|自研或基于开源库定制的解析逻辑|  
### 2.渲染原理
 整体遵循“文本 → 结构化数据 → 目标格式”的编译流程，核心分 3 步：
 1. 词法分析（Lexical Analysis）
 解析器通过正则表达式扫描 Markdown 文本，识别语法元素（如  #  标题、 **  加粗、 >  引用），拆解为令牌（Tokens）。
 - 例： # 标题  被识别为「一级标题令牌」，内容为「标题」
 - 核心组件：Lexer（词法分析器），负责“拆碎片”
 2. 语法分析（Syntactic Analysis）
 将令牌按语法规则排序、嵌套，构建抽象语法树（AST），描述文档结构。
 - 例：标题令牌 + 段落令牌 + 列表令牌，形成层级化的 AST 节点树
 - 核心组件：Parser（语法分析器），负责“理结构”
 3. 渲染输出（Rendering）
 渲染器遍历 AST，将节点转换为目标格式（主流为 HTML），再结合 CSS 呈现视觉效果。
 - 核心组件：Renderer（渲染器），负责“转格式”
