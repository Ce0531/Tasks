## 3.GitHub核心概念  
1. **GitHub是什么？**  
   GitHub 是基于 Git 版本控制系统的全球最大开源代码托管平台，核心作用：
  - 存储、管理代码与文档，支持多人协作开发    
  - 开源项目共享与协作，是程序员的“技术社交网络”    
  - 追踪代码变更、回溯历史版本，保障项目安全
2. **Pull Request和Fork的理解**
  - Fork：将他人仓库复制一份到自己的 GitHub 账号下，相当于“创建个人副本”，可自由修改而不影响原项目
  - Pull Request：修改自己 Fork 的仓库后，向原仓库发起“合并请求”，让原作者审核你的代码并决定是否合并到主项目中

## 4.整洁规范文档的标准      
- 标题层级：使用  #  分级（一级标题  # 、二级  ## 、三级  ### ），逻辑清晰  
- 列表使用：无序列表  - 、有序列表  1.  区分内容类型，避免段落堆砌  
- 代码与强调：用  `  标记技术名词（如  GitHub 、 Pull Request ），用  **加粗**  突出重点  
- 分隔线：用  ---  划分不同模块，提升可读性  
- 格式统一：中英文之间加空格，标点符号使用半角，保持视觉整洁  
## 5.我遵循的规范  
 - 结构清晰：从概念到实践，层层递进，便于快速定位信息  
 - 可读性强：避免大段文字，用列表、分隔线拆分内容，降低阅读压力  
 - 符合技术文档惯例：遵循 GitHub README 通用写法，便于他人协作与维护

# ✨Markdown学习笔记  
### 1. 标题分级    
- 一级标题： # 标题内容 （一个#加空格，字号最大）   
- 二级标题： ## 标题内容 （两个#加空格）  
- 三级标题： ### 标题内容 （三个#加空格）  
以此类推，最多六级标题，#和文字之间必须加空格，不然不生效  
### 2. 无序列表    
规则：每一行开头用  - 空格     
正确效果：每一行都有黑点，文字完全对齐
### 3. 有序列表  
规则：每一行用  数字. 空格  开头，数字后必须加空格，内容直接写，不用额外缩进
### 4. 任务列表  
规则:' - [ ] ' [^02]
[^02]:为了可以展示正确用法，故用‘’括起来  
### 5. 换行/分段  
- 单纯换行（行与行紧挨着）：行尾敲2个空格，再按回车  
- 分段（行与行有空隙）：直接空一行，再写下一段内容  
### 6. 文本简单格式  
- 加粗： **文字内容** （两个星号包裹文字）  
- 斜体： *文字内容* （一个星号包裹文字）
- 粗斜体：***文字内容***（三个星号包裹文字）
- 删除线：~~文字内容~~（两个~~包裹文字）
### 7.链接
- [链接名字](链接地址）
- 拓展：锚点链接（适用于刚开始不知道网址或出现多次同一网址）
  [文字标题][锚点名称]     文章最后输入[锚点名称]：网址   
### 8.引用  
- 单行：>+空格
- 嵌套：多个>，每多一个，更深一层
### 9.脚注  
  文章内容[^序号]   
  [^序号]：脚注内容  
### 10.图片  
- 规则：！[替代文本]（图片路径“可选图片标题”）
- 带链接的图片：将图片的Markdown语法作为链接的“链接文本”部分
### 11.表格  
- 表头：用|来分隔
- 分隔行：用|来分隔，并使用至少三个-来创建
- 左对齐：---：  
  右对齐：：---   
  居中对齐：：---：
- eg:
  | 歌手 |性别|  
  | ---  |---|  
  | 陶喆 |男  |    
### 12.代码块  
- 行内代码：前后各用`来包裹（1前面的按键）
- 代码块：上下各用```来包裹，第一行三个`后面可以指定代码类型
